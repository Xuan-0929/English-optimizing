# 项目结构与整理约定

## 目标

本仓库按“主流程稳定、工具脚本归档、历史脚本隔离”原则组织，避免根目录脚本散落。

## 目录职责

- `train/`
  - 当前生产可用的 SFT 训练流程。
  - 新增训练相关逻辑优先放在这里。

- `scripts/data_pipeline/`
  - 数据生成、扩展、过滤等脚本。
  - 这部分以实验/批处理为主，不作为训练主入口。

- `scripts/evaluation/`
  - 模型评估、对比、数据质量检查脚本。

- `scripts/model_utils/`
  - 模型下载、本地可用性测试等辅助工具。

- `scripts/legacy/`
  - 历史脚本归档区。
  - 默认不在主流程中引用，保留仅用于追溯。

## 运行约定

1. 所有脚本默认从仓库根目录运行。
2. 新脚本不要再放到根目录；按职责放入 `scripts/` 对应子目录。
3. 训练主链路命令优先维护在 `README.md` 中。
4. 若新增目录或迁移脚本，同步更新 `README.md` 与本文件。

## 后续可选优化

1. 给 `scripts/` 下脚本补统一 CLI 参数（`--input/--output`），减少硬编码路径。
2. 增加 `Makefile` 或 `justfile` 作为统一命令入口。
3. 逐步把稳定脚本模块化到 `src/`，便于单元测试和复用。
