# 英语语法纠错数据标注规范（SFT）

## 1. 目标

本规范用于保证训练数据只做“语法纠错”，不做“语义改写”或“风格润色”。
标注输出统一为：

```text
**错误类型**: <类型>
**改正**: <改正后句子>
**解释**: <中文解释>
```

## 2. 核心原则

1. 只改语法，不改语义  
- 不新增事实、人物动机、地点、事件。  
- 不把“没说的内容”补出来。

2. 最小改动原则  
- 仅修改出错片段。  
- 尽量保持原句词序和信息结构。

3. 原句可接受则不强改  
- 如果句子语法可接受，`改正`应与原句一致（或仅做必要标点修正）。  
- 不允许为了“更高级表达”而改写。

4. 改正后必须自然、语法正确  
- 禁止把原句改成更奇怪或错误的句子。

## 3. 错误类型定义（本项目）

- `主谓一致`：主语与谓语数/人称不一致。
- `时态一致`：主从句时态搭配不当、间接引语时态错误。
- `介词`：介词误用/缺失/多余。
- `连接词`：逻辑连接词使用不当（but/so/although 等）。
- `定语从句`：关系代词/结构错误（who/which/that/冗余代词等）。
- `倒装句`：倒装结构缺失或语序错误。
- `不够地道`：语法正确但不符合日常生活使用。

如果一条样本同时有多个错误：
- 优先修正最主要语法错误；
- `错误类型` 填主错误类型，不要混写多个类型。

## 4. 明确禁止（高频脏标注）

1. 语义扩写
- 例如把：`... but my friends didn't want to.`
- 改成：`... but my friends preferred to go to the basketball game.`（新增语义，禁止）

2. 无中生有的对比补写
- 例如随意加入 `rather than ...`，导致信息变多甚至不通顺（禁止）。

3. 把正确句硬改错
- 原句已可接受时，不允许为了凑“纠错”而改写。

4. 风格重写替代纠错
- 例如把 `but` 改成 `despite`、整句重组，只为“更高级”而非纠错（默认禁止，除非原句确有语法错误且必须重构）。

## 5. 各字段要求

### 5.1 `错误类型`
- 必须是上述 7 类之一。
- 与实际修改点一致。

### 5.2 `改正`
- 只输出英文句子本体。
- 不加解释，不加引号，不加编号。
- 保持原句语义不变。

### 5.3 `解释`
- 中文，1-2 句即可。
- 指明“哪里错 + 为什么这样改”。
- 避免空泛描述（如“这样更好”）。

## 6. 判定清单（每条必过）

提交前逐条自检：

1. 改正后是否语法正确且自然？
2. 是否只改了必要部分？
3. 是否引入了原句没有的新信息？
4. 错误类型与修改点是否一致？
5. 解释是否准确对应本次修改？

任一项为“否”，该条需重做。

## 7. 建议的人工修订流程

1. 先看 `input` 与 `current_correction` 是否语义一致。  
2. 若不一致，先重写 `改正`，再回填 `解释`。  
3. 对 `risk_tags` 包含以下标签的样本优先处理：
- `LOW_SIMILARITY`
- `ADDED_RATHER_THAN`
- `ADDED_PREFERRED_TO`
- `TENSE_SUSPICIOUS`

## 8. 输出格式示例

```text
**错误类型**: 时态一致
**改正**: He mentioned he would have dinner with his friend.
**解释**: 间接引语中，主句为过去时，后接从句通常要做时态后移，will 应改为 would。
```
